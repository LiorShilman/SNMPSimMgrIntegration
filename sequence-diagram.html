1. Polling Flow

Logic Object polls hardware via SNMP GET/WALK every X seconds. OidWatchService checks for changes and broadcasts via SignalR.
sequenceDiagram participant Logic as Logic Object (WPF) participant HW as Hardware (SNMP Agent) participant Watch as OidWatchService participant Hub as SnmpHub (SignalR) participant Angular as Angular Panel (Browser) rect rgb(30, 40, 60) Note over Logic,HW: Timer tick - every X seconds Logic->>HW: SNMP GET/WALK (V2c or V3) HW-->>Logic: SNMP Response (OID to Value pairs) Logic->>Logic: Update internal model end rect rgb(40, 50, 70) Note over Logic,Watch: Check OID watches Logic->>Watch: NotifyOidChanged(deviceId, oid, newValue) Watch->>Watch: Compare with previous value alt Value changed and watch registered Watch->>Hub: BroadcastOidChanged(deviceId, oid, value) Hub->>Angular: onOidChanged(deviceId, oid, value) Angular->>Angular: Update field in panel UI else No change or no watch Watch-->>Logic: (no action) end end

2. SET Flow

User changes value in Angular panel. SignalR sends to WPF Hub which performs SNMP SET to hardware and returns result.
sequenceDiagram participant Angular as Angular Panel (Browser) participant Hub as SnmpHub (SignalR) participant Store as DeviceProfileStore participant HW as Hardware (SNMP Agent) participant Watch as OidWatchService Angular->>Hub: sendSet(deviceName, oid, value, type) Hub->>Store: LoadProfilesAsync() Store-->>Hub: List of DeviceProfile Hub->>Hub: Find device by name and active simulator rect rgb(50, 30, 30) Note over Hub,HW: SNMP SET to hardware Hub->>HW: SNMP SET (oid, value, type) to device IP:161 HW-->>Hub: SNMP Response (success/error) end alt SET succeeded Hub->>Store: LoadWalkDataAsync(device) Hub->>Hub: Update walk record with new value Hub->>Store: SaveWalkDataAsync(device, records) Hub-->>Angular: setResult success true Hub->>Watch: NotifyOidChanged(deviceId, oid, value) Watch->>Hub: BroadcastOidChanged (if watched) Hub->>Angular: onOidChanged(deviceId, oid, value) else SET failed Hub-->>Angular: setResult success false with error end

3. Bulk SET Flow

Angular sends multiple SET operations at once. Hub performs SNMP SET for each OID sequentially and returns summary.
sequenceDiagram participant Angular as Angular Panel participant Hub as SnmpHub participant HW as Hardware Angular->>Hub: sendBulkSet(deviceName, items list) loop For each OID in bulk list Hub->>HW: SNMP SET (oid, value, type) HW-->>Hub: Response Hub->>Hub: Track success/failure end Hub-->>Angular: bulkSetResult with results array Hub->>Angular: onOidChanged (for each changed OID)

4. Schema Loading

When user selects a device in the panel, Angular requests schema from Hub. Hub builds schema from MIB files + walk data.
sequenceDiagram participant Angular as Angular Panel participant Hub as SnmpHub participant Store as DeviceProfileStore participant Parser as MibParserService participant Export as MibPanelExportService participant Cache as Schema Cache Angular->>Hub: requestSchema(deviceName) Hub->>Cache: Check cached schema alt Schema in cache Cache-->>Hub: Cached MibPanelSchema Hub-->>Angular: schema(MibPanelSchema) else Not cached Hub->>Store: LoadProfilesAsync() Store-->>Hub: DeviceProfile with MibFilePaths Hub->>Store: LoadWalkDataAsync(device) Store-->>Hub: List of SnmpRecord Hub->>Parser: Parse MIB files Parser-->>Hub: List of MibDefinition Hub->>Export: BuildSchema(device, definitions, walkData) Export-->>Hub: MibPanelSchema Hub->>Cache: Store schema Hub-->>Angular: schema(MibPanelSchema) end Angular->>Angular: Render panel fields with current values

5. OID Watch - Server Automation

Register watches on specific OIDs (exact) or subtrees (prefix). When values change, OidWatchService triggers callbacks and SignalR broadcast.
sequenceDiagram participant Logic as Logic Object participant Watch as OidWatchService participant Hub as SnmpHub participant Angular as Angular Panel rect rgb(60, 50, 20) Note over Logic,Watch: Registration Phase Logic->>Watch: RegisterWatch(deviceId, oid, callback) Watch->>Watch: Store watch entry with match type end rect rgb(30, 50, 40) Note over Logic,Watch: Runtime - on every poll Logic->>Watch: NotifyOidChanged(deviceId, oid, value) Watch->>Watch: Find matching watches (exact OR prefix) alt Has matching watches Watch->>Logic: Execute callback(oid, value) for server automation Watch->>Hub: BroadcastOidChanged(deviceId, oid, value) Hub->>Angular: onOidChanged(deviceId, oid, value) Angular->>Angular: Execute subscribed handlers for client automation end end rect rgb(40, 30, 50) Note over Angular,Hub: Client-side subscription Angular->>Hub: Connection established Note over Angular: oidWatchService.subscribe(deviceId, oid, handler) Note over Angular: handler runs on every onOidChanged matching oid end

6. IDD SET Flow

IDD panel allows setting values by field names (fieldId) instead of direct OID. Hub delegates to SimulatorService which triggers the appropriate Logic.
sequenceDiagram participant Angular as Angular Panel participant Hub as SnmpHub participant Sim as ISimulatorService participant Logic as Logic Object participant HW as Hardware Angular->>Hub: sendIddSet(deviceName, fieldId, value) Hub->>Hub: Find active simulator by device name Hub->>Sim: RaiseIddSet(deviceId, fieldId, value) Sim->>Logic: Handle IDD SET event Logic->>HW: SNMP SET (resolved OID, value) HW-->>Logic: Response Logic->>Logic: Update internal model Note over Hub,Angular: Next poll cycle picks up the change via OidWatch
