2. טעינת Schema — Device Selection STARTUP

כאשר המשתמש בוחר device ב-Angular, נטענת הסכמה בסדר עדיפות: 1) מקובץ JSON מוכן (SchemaPath), 2) בניית IDD מ-IddFields, 3) פענוח MIB files. כולל cache.

3. SNMP SET — Angular → Hardware CLIENT → HW

המשתמש משנה ערך בפאנל Angular. הערך נשלח דרך SignalR ל-WPF Hub, שמבצע SNMP SET לחומרה דרך הפורט של הסימולטור. תוצאת ה-SET חוזרת ל-Angular ומעדכנת את הפאנל.
sequenceDiagram participant User as User (Browser) participant Angular as Angular Panel participant Panel as MibPanelService participant SRS as SignalR Service participant Hub as SnmpHub participant Store as DeviceProfileStore participant Rec as SnmpRecorderService participant HW as Hardware (SNMP Agent) participant Watch as OidWatchService User->>Angular: Change field value in panel Angular->>Panel: sendSet(oid, name, value, valueType) Panel->>Panel: Create SetFeedback (status=pending) Panel->>Panel: Detect SNMP (OID starts with digit) Panel->>SRS: sendSet(deviceId, oid, value, valueType) SRS->>Hub: SendSet(deviceId, oid, value, valueType) Hub->>Store: LoadProfilesAsync() Store-->>Hub: DeviceProfile Hub->>Hub: Find ActiveSimulator by deviceId rect rgb(50, 30, 30) Note over Hub,HW: SNMP SET to hardware via simulator port Hub->>Rec: SetAsync(target 127.0.0.1:simPort, oid, value, type) Rec->>HW: SNMP SET PDU (oid, value) HW-->>Rec: SNMP Response (success/error) Rec-->>Hub: bool success end alt SET succeeded Hub-->>SRS: SetResult(success=true) SRS-->>Panel: result Panel->>Panel: updateFieldValue(oid, value) Panel->>Panel: SetFeedback status=success Panel->>Angular: Update UI field value Note over Hub,Watch: Track change for automation Hub->>Watch: NotifyChange(oid, value) Watch->>Watch: Compare with previous value Watch->>Watch: Fire WatchByName callbacks (if registered) Watch->>Hub: Return previousValue Hub->>Hub: BroadcastOidChanged(deviceId, name, oid, value, prev, source) Hub->>Angular: onOidChanged event Angular->>Angular: Dispatch name-based watches else SET failed Hub-->>SRS: SetResult(success=false, message) SRS-->>Panel: result Panel->>Panel: SetFeedback status=error end Note over Panel: Auto-remove feedback after 8 seconds

4. IDD SET — Angular → Hardware CLIENT → HW

שדות IDD נשלחים לפי שם שדה (לא OID מספרי). ה-Hub מעביר ל-ISimulatorService שמנתב ל-Device Handler המתאים. ה-Handler שולח פקודה לחומרה, ואחרי אישור — קורא ל-NotifyIddSet שמשדר בחזרה ל-Angular.

5. SNMP Poll — Hardware → Angular HW → CLIENT

ה-Logic Object מבצע polling תקופתי (SNMP GET/WALK) לחומרה. התוצאות מועברות דרך IntegrationWiring ל-OidWatch (לזיהוי שינויים) ול-SignalR (לעדכון Angular). שני מסלולים: NotifyTraffic (לכל OID בודד) ו-NotifyWalkResults (ל-batch מלא).
sequenceDiagram participant Timer as DispatcherTimer participant Logic as Logic Object (WPF) participant HW as Hardware (SNMP Agent) participant Wiring as IntegrationWiring participant Watch as OidWatchService participant Hub as SnmpHub participant Angular as Angular Panel rect rgb(40, 40, 30) Note over Timer,HW: Option A — Single OID polling Timer->>Logic: Timer tick (every X seconds) Logic->>HW: SNMP GET (specific OID) HW-->>Logic: Response: OID = newValue Logic->>Wiring: NotifyTraffic(deviceName, "GET", oid, value, sourceIp) Wiring->>Hub: BroadcastTraffic(deviceName, "GET", oid, value) Hub->>Angular: onTrafficReceived Wiring->>Watch: NotifyChange(oid, value) Watch->>Watch: Compare with previous value alt Value changed Watch->>Watch: Fire exact/prefix/name callbacks Watch-->>Wiring: previousValue (different) Wiring->>Hub: BroadcastOidChanged(deviceId, name, oid, value, prev) Hub->>Angular: onOidChanged Angular->>Angular: Update matching field in panel end end rect rgb(30, 40, 50) Note over Timer,Angular: Option B — Full WALK polling (batch) Timer->>Logic: Timer tick Logic->>HW: SNMP WALK (entire subtree) HW-->>Logic: Response: Dictionary of OID to Value Logic->>Wiring: NotifyWalkResults(deviceId, deviceName, walkResults) Note over Wiring,Hub: Step A — Batch broadcast (one SignalR call) Wiring->>Hub: BroadcastMibUpdate(deviceId, walkResults) Hub->>Angular: onMibUpdated (all values at once) Angular->>Angular: Match each OID/name and update fields Note over Wiring,Watch: Step B — Per-OID change detection loop For each OID in walkResults Wiring->>Watch: NotifyChange(oid, value) Watch->>Watch: Compare with previous alt Value changed Watch->>Watch: Fire WatchByName/exact/prefix callbacks Wiring->>Hub: BroadcastOidChanged(deviceId, name, oid, value, prev, "walk") Hub->>Angular: onOidChanged end end end Note over Angular: Angular matches fields by OID or by field.name

6. IDD Update — Hardware → Angular HW → CLIENT

חומרה שולחת עדכון סטטוס ל-WPF (דרך TCP, Serial, Modbus וכו'). ה-Device Handler מקבל את הערך ומעדכן את Angular דרך UpdateField (שדה בודד) או UpdateFields (batch). ה-Angular מזהה את השדה לפי field.name (לא לפי OID מספרי).
sequenceDiagram participant HW as Hardware (TCP/Serial) participant Logic as WPF Event Handler participant Device as SampleIddDevice participant Wiring as IntegrationWiring participant Watch as OidWatchService participant Hub as SnmpHub participant Angular as Angular Panel rect rgb(50, 40, 20) Note over HW,Device: Option A — Single field update HW->>Logic: Status callback (e.g. temperature = 73) Logic->>Device: UpdateField("idd.info.temperature", "73") Device->>Wiring: NotifyIddSet(DeviceId, "idd.info.temperature", "73") Wiring->>Hub: BroadcastTraffic("IDD", "SET", "idd.info.temperature", "73") Hub->>Angular: onTrafficReceived Wiring->>Watch: NotifyChange("idd.info.temperature", "73") Watch->>Watch: Compare: was "42" now "73" — CHANGED Watch->>Watch: Fire WatchByName("Temperature") callbacks Note over Watch: e.g. threshold check — temp > 80? Watch-->>Wiring: previousValue = "42" Wiring->>Hub: BroadcastOidChanged(deviceId, "IDD", "idd.info.temperature", "73", "42") Hub->>Angular: onOidChanged(fieldName="Temperature") Wiring->>Hub: BroadcastMibUpdate(deviceId, dict idd.info.temperature=73) Hub->>Angular: onMibUpdated Angular->>Angular: Match: field.name === "idd.info.temperature" Angular->>Angular: Update panel: Temperature = 73 C end rect rgb(30, 45, 50) Note over HW,Angular: Option B — Batch update (multiple fields) HW->>Logic: Periodic poll returns multiple values Logic->>Device: UpdateFields(dict with 3 fields) Note over Device: temperature=73, uptime=172800, hwStatus=1 Device->>Wiring: NotifyIddBatch(DeviceId, fieldValues) Note over Wiring,Hub: Step A — One batch broadcast Wiring->>Hub: BroadcastMibUpdate(deviceId, 3 fields) Hub->>Angular: onMibUpdated (all 3 values) Angular->>Angular: Match by values[field.name] for each field Note over Wiring,Watch: Step B — Per-field change detection loop For each field in batch Wiring->>Watch: NotifyChange(fieldId, value) alt Value changed Watch->>Watch: Fire WatchByName callbacks Wiring->>Hub: BroadcastOidChanged(deviceId, fieldId, value, prev, "hardware") Hub->>Angular: onOidChanged end end Angular->>Angular: All 3 fields updated in panel end

7. WatchByName — Automation Chain

דוגמה מלאה: חומרה מדווחת על טמפרטורה גבוהה → WatchByName callback ב-WPF מזהה חריגה → שולח UpdateField לשנות מצב alarm → Angular מעדכן את שני השדות (temperature + alarm status). שרשרת אוטומציה מלאה בשתי הכיוונים.
sequenceDiagram participant HW as Hardware participant Logic as WPF Handler participant Device as SampleIddDevice participant Wiring as IntegrationWiring participant Watch as OidWatchService participant Hub as SnmpHub participant Angular as Angular Panel Note over HW,Angular: Scenario: Temperature exceeds threshold — auto-trigger alarm rect rgb(50, 30, 20) Note over HW,Wiring: Step 1 — Hardware reports temperature = 85 HW->>Logic: Temperature sensor callback: 85 C Logic->>Device: UpdateField("idd.info.temperature", "85") Device->>Wiring: NotifyIddSet(deviceId, "idd.info.temperature", "85") Wiring->>Hub: BroadcastMibUpdate + BroadcastOidChanged Hub->>Angular: onMibUpdated + onOidChanged Angular->>Angular: Temperature field updates to 85 C end rect rgb(60, 50, 20) Note over Wiring,Watch: Step 2 — WatchByName fires automation Wiring->>Watch: NotifyChange("idd.info.temperature", "85") Watch->>Watch: Resolve name: "idd.info.temperature" matches "Temperature" Watch->>Watch: Fire WatchByName("temperature") callback Note over Watch: Callback checks: 85 > 80 — ALERT! end rect rgb(30, 50, 30) Note over Watch,Angular: Step 3 — Automation triggers alarm update Watch->>Wiring: NotifyIddSet(deviceId, "idd.info.hwStatus", "0") Note over Wiring: hwStatus 0 = Fault (LED turns red) Wiring->>Hub: BroadcastTraffic("IDD", "SET", "idd.info.hwStatus", "0") Wiring->>Watch: NotifyChange("idd.info.hwStatus", "0") Wiring->>Hub: BroadcastOidChanged(deviceId, "idd.info.hwStatus", "0", "1") Wiring->>Hub: BroadcastMibUpdate(deviceId, hwStatus=0) Hub->>Angular: onMibUpdated + onOidChanged Angular->>Angular: Hardware Status LED turns RED (Fault) end Note over Angular: Result: User sees temperature=85 AND red Fault LED Note over Angular: Both updated automatically — no user interaction needed rect rgb(30, 40, 50) Note over HW,Angular: Later: Temperature drops below 60 — auto-clear alarm HW->>Logic: Temperature = 55 C Logic->>Device: UpdateField("idd.info.temperature", "55") Device->>Wiring: NotifyIddSet(...) Wiring->>Watch: NotifyChange(...) Watch->>Watch: WatchByName fires: 55 <= 60 — clear alarm Watch->>Wiring: NotifyIddSet(deviceId, "idd.info.hwStatus", "1") Wiring->>Hub: Broadcast all events Hub->>Angular: Temperature=55, Status=OK (green LED) end
